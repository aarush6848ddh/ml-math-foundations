Span, Linear Combinations, and Basis

From 3Blue1Brown, Essence of Linear Algebra, Chapter 2
Linear Algebra Lesson 02
Concept

Linear Combinations and Span

Given two vectors \(\mathbf{v}\) and \(\mathbf{w}\), a linear combination is any vector you can make by scaling each one and adding the results: \(a\mathbf{v} + b\mathbf{w}\), where \(a\) and \(b\) are any real numbers.

The span of a set of vectors is the collection of all possible linear combinations. It is the complete set of places you can reach using those vectors as "movement ingredients."

Two independent vectors span the whole plane

If \(\mathbf{v}\) and \(\mathbf{w}\) point in genuinely different directions, you can reach any point in 2D space by choosing the right scalars. Changing \(a\) and \(b\) freely lets the tip of \(a\mathbf{v} + b\mathbf{w}\) sweep the entire plane.

Dependent vectors are stuck on a line

If one vector is just a scaled version of the other, say \(\mathbf{w} = 3\mathbf{v}\), then \(a\mathbf{v} + b\mathbf{w} = a\mathbf{v} + 3b\mathbf{v} = (a + 3b)\mathbf{v}\). No matter what \(a\) and \(b\) you pick, you only ever produce multiples of \(\mathbf{v}\). You are locked onto a single line through the origin.

Linear independence

Two vectors are linearly independent if neither is a scalar multiple of the other. Geometrically, they point in different directions. Algebraically, the only solution to \(a\mathbf{v} + b\mathbf{w} = \mathbf{0}\) is \(a = b = 0\).

The standard basis

The most natural choice of independent vectors in 2D is \(\hat{\imath} = \begin{bmatrix}1\\0\end{bmatrix}\) and \(\hat{\jmath} = \begin{bmatrix}0\\1\end{bmatrix}\). Every 2D vector is already written as a linear combination of these: \(\begin{bmatrix}5\\-3\end{bmatrix} = 5\hat{\imath} + (-3)\hat{\jmath}\). The coordinates of a vector are just its scalars in the standard basis.

Formulas

Key Formulas

Linear combination
\[ a\mathbf{v} + b\mathbf{w}, \quad a,b \in \mathbb{R} \]
Span
\[ \text{span}(\mathbf{v}, \mathbf{w}) = \{ a\mathbf{v} + b\mathbf{w} \mid a,b \in \mathbb{R} \} \]
Solving for scalars
\[ \begin{bmatrix} \mathbf{v} & \mathbf{w} \end{bmatrix} \begin{bmatrix} a \\ b \end{bmatrix} = \mathbf{t} \]
2D cross product (dependence test)
\[ \mathbf{v} \times \mathbf{w} = v_1 w_2 - v_2 w_1 \]

If the cross product equals zero, the vectors are linearly dependent and their span is only a line. If it is nonzero, they are independent and their span is the full plane.

Code

Code Examples

All code below is taken verbatim from lesson02-span.py.

Part 1: Linear combinations

Two linearly independent vectors \(\mathbf{v} = \begin{bmatrix}2\\3\end{bmatrix}\) and \(\mathbf{w} = \begin{bmatrix}1\\4\end{bmatrix}\). Trying several scalar pairs shows the results pointing in many different directions, confirming the vectors span the plane.

Python lesson02-span.py 
import numpy as np

# define two linearly independent basis vectors
v = np.array([2, 3])
w = np.array([1, 4])

# a*v + b*w = some vector in the plane
scalar_pairs = [(1,1), (2,-1), (0,3), (-1,2), (3,3)]
for a, b in scalar_pairs:
    result = (a * v) + (b * w)
    print(f"a = {a}, b = {b} -> {result}")

Part 2: Reaching any target with independent vectors

Since \(\mathbf{v}\) and \(\mathbf{w}\) are independent, we can reach any target point by solving the linear system for the right scalars. np.linalg.solve finds the coefficients \(a\) and \(b\) such that \(a\mathbf{v} + b\mathbf{w} = \mathbf{t}\).

Python lesson02-span.py 
# two linearly independent vectors can reach any point (using vectors from part 1)
matrix = np.column_stack((v, w))  # columns are the basis vectors
target = np.array([7, 4])

# find the scalars needed such that a*v + b*w = target
scalars = np.linalg.solve(matrix, target)
print(f"Scalars to reach {target}: {scalars}")

Part 3: Dependent vectors are stuck on a line

When \(\mathbf{w}_{dep} = 3\mathbf{v}_{dep}\), every linear combination \(a\mathbf{v}_{dep} + b\mathbf{w}_{dep}\) collapses to a scalar multiple of \(\mathbf{v}_{dep}\). Notice in the output that the second component is always exactly 3 times the first. The two vectors have the same span as one.

Python lesson02-span.py 
# two linearly DEPENDENT vectors can only span a line
# one is just a scaled version of the other
v_dep = np.array([1, 3])
w_dep = np.array([3, 9])

for a, b in scalar_pairs:
    result = (a * v_dep) + (b * w_dep)
    print(f"Dependent combo: a = {a}, b = {b} -> {result}")

# Output: second number is always 3x the first.
# No matter which scalars you pick, you stay on the same line through the origin.

Part 4: Checking linear dependence with the cross product

In 2D, the cross product \(v_1 w_2 - v_2 w_1\) measures the area of the parallelogram formed by the two vectors. Area zero means they are collinear, i.e., dependent. NumPy's np.cross computes this directly.

Python lesson02-span.py 
def are_linearly_dependent(v, w) -> bool:
    if np.cross(v, w) == 0:
        return True
    else:
        return False

# (2*4) - (1*3) = 5, nonzero: independent
print(are_linearly_dependent(v, w))      # False

# (1*9) - (3*3) = 0: dependent
print(are_linearly_dependent(v_dep, w_dep))  # True

Part 5: Standard basis vectors

\(\hat{\imath}\) and \(\hat{\jmath}\) are the simplest independent pair. With the standard basis, solving for scalars is trivial because the matrix is the identity. The scalars are just the coordinates of the target.

Python lesson02-span.py 
# i_hat and j_hat are the building blocks of all 2D space
i_hat = np.array([1, 0])
j_hat = np.array([0, 1])

target_point = np.array([5, -3])

# to reach [5, -3] just scale i_hat by 5 and j_hat by -3
matrix2 = np.column_stack((i_hat, j_hat))
result = np.linalg.solve(matrix2, target_point)
print(f"Reaching {target_point} using standard basis: {result}")
AI Connection

How This Connects to AI

Why this matters for machine learning

In a transformer, each token is a vector in a high-dimensional embedding space. That space has a basis: a set of independent directions that can reach any point in it. The model does not use \(\hat{\imath}\) and \(\hat{\jmath}\). It learns its own basis, the directions most useful for representing language.

Every embedding dimension is like a basis vector. The 768 dimensions in BERT or the 12,288 dimensions in GPT-3 are the coordinates of each token in the model's learned basis. Training adjusts those basis directions so that meaningful concepts end up as consistent, reachable locations in the space.

Span is why high-dimensional spaces are powerful. With 768 independent directions, the span is all of \(\mathbb{R}^{768}\). Every possible nuance of meaning has a place to live. If the dimensions were dependent (redundant), the model would be stuck on a lower-dimensional subspace and could not represent the full richness of language.

Linear dependence shows up as a failure mode. When attention heads in a transformer learn redundant directions, they waste capacity. Research like "Attention is Not All You Need" studies exactly this: how many of the learned basis directions are truly independent and necessary.
