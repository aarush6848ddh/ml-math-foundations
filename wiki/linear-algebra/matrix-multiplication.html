Matrix Multiplication as Composition

From 3Blue1Brown, Essence of Linear Algebra, Chapter 4
Linear Algebra Lesson 04
Concept

Multiplying Matrices Means Composing Transformations

When you multiply two matrices together, you are not doing some arbitrary number crunching. You are asking: what single transformation has the same effect as applying one transformation and then another? That combined transformation is called the composition, and the resulting matrix is the product.

The key insight: \(AB\mathbf{v}\) means "apply \(B\) first, then apply \(A\)." Read right to left. The product \(AB\) is the single matrix that does both steps at once.

Order matters

Matrix multiplication is not commutative. Rotating then shearing produces a different result than shearing then rotating. \(AB \neq BA\) in general. The geometric reason is simple: the first transformation changes the space, and the second transformation acts on that already-changed space. Reverse the order and you get a different chain of events.

The exception: uniform scaling

A scaling matrix. \(cI\) (the identity scaled by a constant) commutes with every matrix. Geometrically, uniform scaling stretches space equally in all directions, so it does not interact with any other transformation. It does not matter whether you scale before or after rotating: the result is the same.

Associativity

Matrix multiplication is associative: \((AB)C = A(BC)\). The grouping of parentheses does not change the final result, as long as the order stays the same. This is not obvious from the arithmetic, but geometrically it makes sense: applying three transformations in sequence produces the same composition no matter how you group the steps.

Formulas

Key Formulas

Composition: apply B then A
\[ (AB)\mathbf{v} = A(B\mathbf{v}) \]
Non-commutativity (in general)
\[ AB \neq BA \]
Scaling commutes with everything
\[ (cI)A = A(cI) = cA \]
Associativity
\[ (AB)C = A(BC) \]
Code

Code Examples

All code below is taken verbatim from lesson-04-matrix-multiplication.py.

Part 1: Composition, the long way vs. the short way

Rotate a vector, then shear the result. Two separate matmul calls or one composition matrix: same answer.

Python lesson-04-matrix-multiplication.py 
import numpy as np

# rotation matrix
rotation_matrix = np.array([
    [0, -1],
    [1,  0]
])

# shear matrix
shear_matrix = np.array([
    [1, 1],
    [0, 1]
])

v = np.array([2, 1])

# apply rotation first then shear the long way
step1 = np.matmul(rotation_matrix, v)
step2 = np.matmul(shear_matrix, step1)
print(f"Long way: {step2}")

# compute the composition matrix in one shot
composition = np.matmul(shear_matrix, rotation_matrix)
result = np.matmul(composition, v)
print(f"Composition: {result}")
Output
Long way: [1 2]
Composition: [1 2]

Part 2: Order matters (non-commutativity)

Rotate then shear is not the same as shear then rotate. The two composition matrices are different, proving \(AB \neq BA\).

Python lesson-04-matrix-multiplication.py 
rotation_shear = np.matmul(rotation_matrix, shear_matrix)
shear_rotation = np.matmul(shear_matrix, rotation_matrix)
print(f"rotation @ shear: {rotation_shear}")
print(f"shear @ rotation: {shear_rotation}")
Output
rotation @ shear: [[ 0 -1]
 [ 1  1]]
shear @ rotation: [[ 1 -1]
 [ 1  0]]

Part 3: The exception: scaling commutes with everything

A uniform scaling matrix is the identity times a scalar. Because it stretches equally in every direction, the order with any other matrix does not matter.

Python lesson-04-matrix-multiplication.py 
# identity matrix scaled by 5
scaling_matrix = np.array([
    [5, 0],
    [0, 5]
])

normal_matrix = np.array([
    [1, 5],
    [3, 5]
])

composition1 = np.matmul(scaling_matrix, normal_matrix)
composition2 = np.matmul(normal_matrix, scaling_matrix)
print(f"scaling @ normal: {composition1}")
print(f"normal @ scaling: {composition2}")
print(f"Do they commute? {np.array_equal(composition1, composition2)}")
# A scaling matrix is just the identity times a scalar.
# The identity does nothing, so it commutes with everything.
Output
scaling @ normal: [[ 5 25]
 [15 25]]
normal @ scaling: [[ 5 25]
 [15 25]]
Do they commute? True

Part 4: Associativity

\((AB)C\) and \(A(BC)\) always give the same result. The grouping does not matter, only the order does.

Python lesson-04-matrix-multiplication.py 
a = np.array([[3, 4], [5, 7]])
b = np.array([[5, 8], [3, 2]])
c = np.array([[9, 6], [7, 1]])

computation1 = np.matmul(np.matmul(a, b), c)
computation2 = np.matmul(a, np.matmul(b, c))
print(f"Are they the same? {np.array_equal(computation1, computation2)}")
Output
Are they the same? True
AI Connection

How This Connects to AI

Why this matters for machine learning

Every forward pass through a neural network is matrix composition. Each layer applies a transformation and the result flows into the next layer. The order of layers is non-negotiable, just like matrix order matters. You cannot swap the attention block and the feedforward block and expect the same result.

Associativity makes deep networks computationally efficient. When you see x @ W1 @ W2 @ W3 in a transformer, it does not matter whether you compute (x @ W1 @ W2) @ W3 or x @ (W1 @ W2 @ W3). The result is identical. GPUs exploit this to batch and parallelize matrix operations in any grouping that is most efficient for the hardware.

Non-commutativity is why architecture order is sacred. In a transformer: query, key, value projections happen before attention. Attention happens before the feedforward layer. Layer norm goes in a specific place. Each of these is a matrix multiplication, and swapping their order would produce completely different outputs. Non-commutativity is the mathematical reason architecture decisions are not interchangeable.

Uniform scaling is why weight initialization uses identity-like matrices. Scaling commutes because it is just the identity times a constant. Early in training, initializing weights close to the identity keeps the signal flowing through many layers without exploding or vanishing. The identity matrix is the neutral element of composition: it does nothing, and it interacts cleanly with whatever comes next.
