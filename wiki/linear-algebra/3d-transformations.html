Extending Transformations to 3D

From 3Blue1Brown, Essence of Linear Algebra, Chapter 5
Linear Algebra Lesson 05
Concept

Everything from 2D, One Dimension Up

Everything covered in 2D extends directly to 3D. A 3D matrix is still a collection of column vectors telling you where each basis vector lands. Matrix-vector multiplication still works the same way. The only change is that there are now three basis vectors: \(\hat{i}\), \(\hat{j}\), and \(\hat{k}\).

The key insight: The columns of a 3D transformation matrix are exactly where \(\hat{i}\), \(\hat{j}\), and \(\hat{k}\) land after the transformation. Same rule as 2D, just one more column and one more row.

Reading a 3D matrix

A 3x3 matrix has three columns. Column one is where \(\hat{i} = (1,0,0)\) lands. Column two is where \(\hat{j} = (0,1,0)\) lands. Column three is where \(\hat{k} = (0,0,1)\) lands. To transform any vector, express it as a combination of the basis vectors and apply the same linear combination to the transformed columns.

Rotation around the z-axis

A 90 degree counterclockwise rotation around the z-axis leaves the z-axis untouched. In the xy-plane it behaves exactly like the 2D rotation: \(\hat{i}\) moves to where \(\hat{j}\) was, and \(\hat{j}\) moves to the negative x direction. \(\hat{k}\) stays at \((0,0,1)\) because the z-axis is the axis of rotation.

Formulas

Key Formulas

3D matrix-vector multiplication
\[ \begin{bmatrix} a & b & c \\ d & e & f \\ g & h & i \end{bmatrix} \begin{bmatrix} x \\ y \\ z \end{bmatrix} = x\begin{bmatrix} a \\ d \\ g \end{bmatrix} + y\begin{bmatrix} b \\ e \\ h \end{bmatrix} + z\begin{bmatrix} c \\ f \\ i \end{bmatrix} \]
90° counterclockwise rotation around z-axis
\[ R_z(90°) = \begin{bmatrix} 0 & -1 & 0 \\ 1 & 0 & 0 \\ 0 & 0 & 1 \end{bmatrix} \]
Code

Code Examples

All code below is taken verbatim from lesson-05-3d-transformations.py.

Part 1: 3D matrix-vector multiplication

Same np.matmul call as in 2D. NumPy handles any number of dimensions.

Python lesson-05-3d-transformations.py 
import numpy as np

# 3D matrix
matrix = np.array([
    [0,   0.5, -0.5],
    [0,   0.5,  1  ],
    [1,   0,    0.5]
])

# 3D vector
v = np.array([1, 0, -2])

# resulting output vector
result = np.matmul(matrix, v)
print(f"Resulting output vector: {result}")
Output
Resulting output vector: [ 1. -2.  0.]

Part 2: 3D rotation matrix

Verify that each basis vector lands exactly where the geometry says it should after a 90 degree rotation around the z-axis.

Python lesson-05-3d-transformations.py 
# 90 degree counterclockwise rotation around the z-axis
# the columns tell you where each basis vector lands
rotation_3d = np.array([
    [0, -1, 0],
    [1,  0, 0],
    [0,  0, 1]
])

# verify each basis vector lands where the lesson says it should
i_hat = np.array([1, 0, 0])
j_hat = np.array([0, 1, 0])
k_hat = np.array([0, 0, 1])

print(f"i_hat lands at: {np.matmul(rotation_3d, i_hat)}")
print(f"j_hat lands at: {np.matmul(rotation_3d, j_hat)}")
print(f"k_hat lands at: {np.matmul(rotation_3d, k_hat)}")
Output
i_hat lands at: [0 1 0]
j_hat lands at: [-1  0  0]
k_hat lands at: [0 0 1]
AI Connection

How This Connects to AI

Why this matters for machine learning

Everything covered in 2D and 3D works identically in any number of dimensions. GPT-4 uses 12,288-dimensional vectors for each token. Every attention operation and weight matrix in a transformer is this same operation, matrix-vector multiplication, running in very high dimensions. The dimension changes. The concept does not.

When you see embedding_size=512 in a transformer config, that is a 512-dimensional version of exactly what was coded here. The intuition built in 2D and 3D is the intuition that applies to every embedding, every projection, and every weight matrix in modern deep learning.
