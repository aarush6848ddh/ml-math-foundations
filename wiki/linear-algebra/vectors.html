Vectors

From 3Blue1Brown, Essence of Linear Algebra, Chapter 1
Linear Algebra Lesson 01
Concept

What is a Vector?

There are three ways to think about a vector, depending on who is in the room:

Physics student: A vector is an arrow in space with a length and a direction. You can move it anywhere and it stays the same vector.
CS student: A vector is an ordered list of numbers, coordinates that describe a point relative to some origin.
3Blue1Brown's view: Think geometrically first, then let the numbers follow. Every list of numbers corresponds to an arrow rooted at the origin.

The coordinate picture

In 2D, a vector like \(\mathbf{v} = \begin{bmatrix} 3 \\ -2 \end{bmatrix}\) means: "walk 3 units right, then 2 units down." The tip of that arrow is the vector. In \(n\) dimensions, a vector is just an ordered list of \(n\) numbers.

Two fundamental operations

Everything in linear algebra is built from just two operations: vector addition and scalar multiplication.

Addition: place two arrows tip-to-tail. The result is a new arrow from the origin to the final tip. Numerically, add component by component.

Scalar multiplication: stretching, shrinking, or flipping an arrow. Multiplying by 2 doubles the length. Multiplying by −1 reverses direction. Multiplying by a number between 0 and 1 squishes it.

Formulas

Key Formulas

Vector notation
\[ \mathbf{v} = \begin{bmatrix} v_1 \\ v_2 \\ \vdots \\ v_n \end{bmatrix} \]
Vector addition
\[ \mathbf{a} + \mathbf{b} = \begin{bmatrix} a_1 + b_1 \\ a_2 + b_2 \\ \vdots \end{bmatrix} \]
Scalar multiplication
\[ c \cdot \mathbf{v} = \begin{bmatrix} c \, v_1 \\ c \, v_2 \\ \vdots \end{bmatrix} \]
Vector subtraction
\[ \mathbf{a} - \mathbf{b} = \mathbf{a} + (-1)\mathbf{b} \]

Scalar multiplication distributes over addition: \( c(\mathbf{a} + \mathbf{b}) = c\mathbf{a} + c\mathbf{b} \). This property, linearity, is what gives linear algebra its name.

Code

Code Examples

All code below is taken verbatim from lesson-01-vectors.py. NumPy represents vectors as 1D arrays and applies all operations element-wise.

Setting up word embeddings as vectors

We model each word as a NumPy array. The four numbers in each array are its coordinates in a 4-dimensional embedding space. In real models these arrays have hundreds or thousands of dimensions, but the idea is identical.

Python lesson-01-vectors.py 
import numpy as np

# A word embedding in AI is a vector that represents the meaning of a word.
# For example, the word "king" might be represented as a vector in a high-dimensional
# space, where each dimension corresponds to a different aspect of the word's

# Pretend these are simplified embeddings for the words "king", "queen
# Vectors from a Computer Science students view can be seen as an ordered list of numbers
king = np.array([2.1, 0.3, -1.2, 0.8])
queen = np.array([1.9, 0.5, -1.1, 0.9])

Vector addition

Adding two vectors means adding each pair of components. Geometrically, place the second arrow at the tip of the first. The result is the arrow from the origin to the new tip. Here it produces a vector that blends the meaning of both words.

Python lesson-01-vectors.py 
# We can perform vector operations to find relationships between words.

# For example, we can add the vectors for "king" and "queen" to find a new vector
# that represents the concept of royalty.
result = king + queen

# The result can be interpreted as a new vector that captures the combined
# meaning of "king" and "queen". For example: Royalty.
print(result)
Output
[ 4.   0.8 -2.3  1.7]

Scalar multiplication

Multiplying a vector by a number scales the length of the arrow without changing its direction (unless the number is negative, which flips it). A scalar above 1 amplifies the concept; a scalar between 0 and 1 weakens it; -1 reverses it.

Python lesson-01-vectors.py 
# We can also scale the vector for "king" by multiplying it by a scalar to find a new vector
# that represents a stronger or weaker version of the concept. For example: Stronger Royalty or Weaker Royalty.
scaled_up = 1.5 * king
flipped = -1 * king

print("scaled_up:", scaled_up)
print("flipped:", flipped)
Output
scaled_up: [ 3.15  0.45 -1.8   1.2 ]
flipped: [-2.1 -0.3  1.2 -0.8]

Vector subtraction

Subtracting one vector from another gives the arrow pointing from the tip of the second to the tip of the first. In embedding space, this captures the direction that separates two concepts. Here, the gender direction between king and queen.

Python lesson-01-vectors.py 
# We can also find the difference between the vectors for "king" and "queen" to find a new vector
# that represents the relationship between the two words. For example: Gender Difference.
difference = king - queen

# This new vector can be interpreted as capturing the concept of genders.
print("difference:", difference)
Output
difference: [ 0.2 -0.2 -0.1 -0.1]

The famous word embedding analogy

Take the direction that separates "man" from "woman" and apply it to "king". The result lands very close to "queen" in the vector space. This was not programmed in Models like word2vec and GPT discover this geometry by training on text alone.

Python lesson-01-vectors.py 
# The famous word embedding analogy
# king - man + woman = queen
# This is how real LLMs like GPT understand relationships between words and concepts.
# The model learns that "gender direction" in vector space is consistent

man = np.array([1.0, 0.1, -0.5, 0.2])
woman = np.array([0.9, 0.6, -0.4, 0.8])

analogy_result = king - man + woman
print("king - man + woman =", analogy_result)
Output
king - man + woman = [ 2.   0.8 -1.1  1.4]
AI Connection

How This Connects to AI

Why this matters for machine learning

In real LLMs like GPT, every word (or token) is represented as a high-dimensional vector called an embedding. A typical embedding has 768, 1536, or even 12,288 dimensions. The same arithmetic you did above works at that scale.

The famous analogy king − man + woman ≈ queen was discovered in word2vec (2013), not programmed in but learned from text. The model learned that there's a consistent "gender direction" in vector space, and that royalty + gender-flip stays royalty.

Vector addition = combining concepts. When a transformer processes "the tall professor", it builds a vector that blends both meanings through addition-like operations in attention layers.

Scalar multiplication = intensity. Attention weights (the scalars in "scaled dot-product attention") literally scale vectors up or down before summing them. The exact same operation.

Bottom line: every forward pass through a neural network is a cascade of vector additions and scalar multiplications. Lesson 1 is the entire foundation.
