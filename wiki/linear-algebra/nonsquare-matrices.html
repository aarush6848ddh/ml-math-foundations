Non-Square Matrices

From 3Blue1Brown, Essence of Linear Algebra, Chapter 8
Linear Algebra Lesson 08
Concept

Transformations Between Different Dimensions

Every matrix we have seen so far has been square: 2x2 or 3x3. That means the input and output live in the same number of dimensions. But in real systems, and especially in neural networks, you almost always need to change the number of dimensions. That is exactly what a non-square matrix does.

The core idea: An \(m \times n\) matrix transforms a vector from \(\mathbb{R}^n\) to \(\mathbb{R}^m\). The number of columns tells you the input dimension. The number of rows tells you the output dimension.

Mapping up: 2D to 3D

A 3x2 matrix takes a 2D vector and produces a 3D vector. You are lifting a point from a plane into a higher-dimensional space. The two columns of the matrix are the basis vectors of the 2D plane, but expressed as 3D coordinates. The result is a 2D plane embedded inside 3D space.

Mapping down: 3D to 2D

A 2x3 matrix takes a 3D vector and produces a 2D vector. You are projecting down, collapsing one dimension. Information can be lost in this direction: multiple 3D points may land on the same 2D output. The matrix cannot be inverted.

The shape rule

For the multiplication \(Av\) to work, the number of columns in \(A\) must equal the length of \(v\). The output has as many entries as \(A\) has rows. This is the same dimension compatibility rule you use when chaining layers in a neural network.

Formulas

Key Formulas

Shape of output
\[ A_{m \times n} \, v_{n \times 1} = w_{m \times 1} \]
2D to 3D example
\[ \begin{bmatrix} 1 & 0 \\ 3 & 5 \\ 9 & 2 \end{bmatrix} \begin{bmatrix} x \\ y \end{bmatrix} = \begin{bmatrix} x \\ 3x + 5y \\ 9x + 2y \end{bmatrix} \]
3D to 2D example
\[ \begin{bmatrix} a & b & c \\ d & e & f \end{bmatrix} \begin{bmatrix} x \\ y \\ z \end{bmatrix} = \begin{bmatrix} ax + by + cz \\ dx + ey + fz \end{bmatrix} \]
Compatibility rule
\[ \underbrace{A}_{m \times n} \cdot \underbrace{B}_{n \times p} = \underbrace{C}_{m \times p} \]
Code

Code Examples

All code below is taken verbatim from lesson-08-nonsquare-matrices.py.

Part 1: 2D to 3D transformation

A 3x2 matrix takes a 2D input vector and lifts it into 3D space. The output shape changes from (2,) to (3,).

Python lesson-08-nonsquare-matrices.py 
import numpy as np

three_by_two = np.array([
    [1, 0],
    [3, 5],
    [9, 2]
])

v = np.array([1, 4])

t = np.matmul(three_by_two, v)

print(f"Input vector: {v}")
print(f"Input shape: {v.shape}")
print(f"Output vector: {t}")
print(f"Output shape: {t.shape}")
Output
Input vector: [1 4]
Input shape: (2,)
Output vector: [ 1 23 17]
Output shape: (3,)

Part 2: 3D to 2D transformation

A 2x3 matrix takes a 3D input vector and projects it down to 2D. The output shape changes from (3,) to (2,).

Python lesson-08-nonsquare-matrices.py 
two_by_three = np.array([
    [4, 6, 3],
    [7, 2, 9]
])

v2 = np.array([4, 6, 2])

t2 = np.matmul(two_by_three, v2)

print(f"Input vector: {v2}")
print(f"Input shape: {v2.shape}")
print(f"Output vector: {t2}")
print(f"Output shape: {t2.shape}")
Output
Input vector: [4 6 2]
Input shape: (3,)
Output vector: [58 64]
Output shape: (2,)
AI Connection

How This Connects to AI

Why this matters for machine learning

Non-square matrices are the fundamental building block of every neural network. Every linear layer is just an \(m \times n\) matrix multiplying an input vector, changing its dimensionality as data flows through the network.

The embedding layer in a transformer is a perfect example of mapping up. A token ID is a single integer, but the model represents it as a rich 512D or 1024D vector. The embedding matrix is a non-square matrix that lifts each token from a one-hot representation into a high-dimensional space where meaning can be encoded.

The output projection layer at the end of a transformer does the opposite. It takes the high-dimensional hidden state and maps it down to vocabulary size (often 50,000 or more dimensions) so the model can produce a probability distribution over the next token. That is a non-square matrix mapping from high dimension to low dimension.

When you see a layer defined as nn.Linear(512, 2048) in PyTorch, the weight matrix is 2048x512. It maps every 512D input up to 2048D. This is the feed-forward expansion inside a transformer block, and it is just a non-square matrix transformation.
