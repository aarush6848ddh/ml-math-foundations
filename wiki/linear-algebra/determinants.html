Determinants: Measuring How Space Gets Scaled

From 3Blue1Brown, Essence of Linear Algebra, Chapter 6
Linear Algebra Lesson 06
Concept

A Single Number That Captures Everything About Scaling

Every linear transformation stretches, squishes, or flips space in some way. The determinant is the number that tells you by exactly how much. Specifically, it tells you how much a transformation scales areas (in 2D) or volumes (in 3D).

The core idea: If the determinant of a matrix is 3, any region of space gets scaled to 3 times its original area after that transformation. If it is 0.5, regions shrink to half. If it is 0, space gets flattened entirely into a lower dimension.

Zero determinant: rank deficiency

A determinant of zero means the transformation squishes space down into a lower dimension. A 2D matrix with det = 0 collapses the entire plane onto a single line (or point). The columns of that matrix are linearly dependent, which means they lie on the same line. This is exactly what happened with the matrix [[1, 2], [2, 4]] in the code: column 2 is just 2 times column 1, so the transformation flattens the plane.

Negative determinant: orientation flipped

The sign of the determinant tells you whether the transformation flipped the orientation of space. Think of it like flipping a sheet of paper over. As the basis vectors \(\hat{i}\) and \(\hat{j}\) slowly rotate toward each other, the area between them shrinks. When they align completely, the determinant hits zero. If the transformation keeps going past that point, the orientation flips and the determinant goes negative. The absolute value still tells you the scaling factor. The sign just tells you whether space was flipped.

Multiplicative property

If you apply transformation A followed by transformation B, the total scaling is just the product of the individual scalings. This is why \(\det(M_1 \cdot M_2) = \det(M_1) \cdot \det(M_2)\). Applying two transformations in sequence multiplies their scaling factors.

Formulas

Key Formulas

2x2 determinant
\[ \det\begin{bmatrix} a & b \\ c & d \end{bmatrix} = ad - bc \]
Multiplicative property
\[ \det(M_1 \cdot M_2) = \det(M_1) \cdot \det(M_2) \]
Zero determinant means linear dependence
\[ \det(M) = 0 \iff \text{columns of } M \text{ are linearly dependent} \]
Code

Code Examples

All code below is taken verbatim from lesson-06-determinants.py.

Part 1: Computing determinants

Three matrices: a shear (area preserved), a squish (area halved), and a rank-deficient matrix with zero determinant.

Python lesson-06-determinants.py 
import numpy as np

strech_matrix = np.array([
    [1, -1],
    [0,  1]
])

sqiush_matrix = np.array([
    [0.5, 0  ],
    [0,   0.5]
])

no_det_matrix = np.array([
    [1, 2],
    [2, 4]
])

d1 = np.linalg.det(strech_matrix)
d2 = np.linalg.det(sqiush_matrix)
d3 = np.linalg.det(no_det_matrix)

print(f"Stretch matrix determinant: {d1}")
print(f"Squish matrix determinant: {d2}")
print(f"Zero determinant matrix: {d3}")
Output
Stretch matrix determinant: 1.0
Squish matrix determinant: 0.25
Zero determinant matrix: 0.0

Part 2: Negative determinant

A matrix that flips the x-axis. The determinant is -1: scaling factor is 1 (no size change), but orientation is flipped.

Python lesson-06-determinants.py 
neg_d_matrix = np.array([
    [-1, 0],
    [ 0, 1]
])

det = np.linalg.det(neg_d_matrix)
print(f"Determinant of flipped orientation matrix: {det}")

# a negative determinant means the transformation flipped the orientation of space
# as i-hat and j-hat slowly rotate toward each other, the area between them shrinks
# when they line up completely, the determinant hits 0
# if the transformation keeps going past that point, the orientation flips
# and the determinant continues into negative numbers
# the absolute value of the determinant still tells you how much area was scaled
# the negative sign just tells you orientation was flipped
Output
Determinant of flipped orientation matrix: -1.0

Part 3: Multiplicative property

Verifying that applying two transformations in sequence multiplies their determinants.

Python lesson-06-determinants.py 
m1 = np.array([[2, 3], [7, 5]])
m2 = np.array([[5, 4], [9, 0]])

# det(m1 * m2)
det1 = np.linalg.det(np.matmul(m1, m2))

# det(m1) * det(m2)
det2 = np.linalg.det(m1) * np.linalg.det(m2)

print(f"Are they equal?: {np.isclose(det1, det2)}")
Output
Are they equal?: True
AI Connection

How This Connects to AI

Why this matters for machine learning

A zero determinant in a weight matrix means rank deficiency: the layer is squishing input data into a lower-dimensional subspace and losing information permanently. No amount of further training can recover what was lost. This is why weight initialization matters. Random initialization (like Xavier or Kaiming init) is specifically designed to keep matrices full rank so no information is destroyed before training even begins.

The multiplicative property connects directly to the vanishing gradient problem. During backpropagation, gradients flow backwards through every layer as a chain of matrix multiplications. The determinant of the overall product equals the product of each layer's determinant. If each weight matrix slightly squishes space (det slightly less than 1), multiplying many of them together collapses the gradient signal toward zero. This is not a coincidence: it is exactly what the determinant is telling you.
