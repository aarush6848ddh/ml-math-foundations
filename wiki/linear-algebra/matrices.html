Matrices as Linear Transformations

From 3Blue1Brown, Essence of Linear Algebra, Chapter 3
Linear Algebra Lesson 03
Concept

Matrices are Transformations

A matrix is not just a grid of numbers. It describes a linear transformation: a function that takes a vector as input and moves it to a new location, while keeping the origin fixed and preserving straight lines.

The key insight: to fully describe where a linear transformation sends every vector in 2D space, you only need to record where the two basis vectors \(\hat{\imath}\) and \(\hat{\jmath}\) land. Everything else follows automatically from linearity. The columns of the matrix are exactly those landing spots.

Reading a matrix

Given a matrix \(M = \begin{bmatrix} a & b \\ c & d \end{bmatrix}\), the first column \(\begin{bmatrix} a \\ c \end{bmatrix}\) is where \(\hat{\imath}\) lands, and the second column \(\begin{bmatrix} b \\ d \end{bmatrix}\) is where \(\hat{\jmath}\) lands.

Matrix-vector multiplication as a linear combination

To transform a vector \(\mathbf{v} = \begin{bmatrix} x \\ y \end{bmatrix}\), scale each column by the corresponding coordinate and add:

\[ M\mathbf{v} = x \cdot \text{col}_1(M) + y \cdot \text{col}_2(M) \]

This is exactly what np.matmul computes. The column-scaling view is more intuitive than the row-dot-product view for understanding what is geometrically happening.

Rotation, shear, and squish

Different matrices produce different geometric effects. A rotation matrix spins every vector by a fixed angle. A shear matrix tilts space, keeping one axis fixed while sliding the other. A matrix with linearly dependent columns collapses all of 2D space onto a single line. The columns of the matrix completely determine which transformation you get.

Formulas

Key Formulas

Matrix-vector product (column view)
\[ M\mathbf{v} = v_1 \mathbf{c}_1 + v_2 \mathbf{c}_2 \]
90° counterclockwise rotation
\[ R_{90} = \begin{bmatrix} 0 & -1 \\ 1 & 0 \end{bmatrix} \]
Horizontal shear
\[ S = \begin{bmatrix} 1 & 1 \\ 0 & 1 \end{bmatrix} \]
Dependent columns (rank 1, collapses to a line)
\[ D = \begin{bmatrix} 1 & 5 \\ 3 & 15 \end{bmatrix}, \quad \text{col}_2 = 5 \cdot \text{col}_1 \]
Code

Code Examples

All code below is taken verbatim from lesson-03-matrices.py.

Part 1: Applying a transformation matrix to a vector

The matrix \(\begin{bmatrix}1 & 3\\2 & 1\end{bmatrix}\) sends \(\hat{\imath}\) to \(\begin{bmatrix}1\\2\end{bmatrix}\) and \(\hat{\jmath}\) to \(\begin{bmatrix}3\\1\end{bmatrix}\). To transform \(\mathbf{v} = \begin{bmatrix}5\\3\end{bmatrix}\), scale column 1 by 5 and column 2 by 3, then add. NumPy's np.matmul gives the same result.

Python lesson-03-matrices.py 
import numpy as np

# column 1 = where i_hat [1, 0] lands
# column 2 = where j_hat [0, 1] lands
matrix = np.array([
    [1, 3],
    [2, 1]
])

v = np.array([5, 3])

# manual: scale column 1 by v[0], scale column 2 by v[1], add
col1 = matrix[:, 0]
col2 = matrix[:, 1]
manual_result = (v[0] * col1) + (v[1] * col2)
print(f"Manual result: {manual_result}")

numpy_result = np.matmul(matrix, v)
print(f"Numpy result: {numpy_result}")
Output
Manual result: [14 13]
Numpy result: [14 13]

Part 2: Rotation matrix (90 degrees counterclockwise)

A 90-degree rotation sends \(\hat{\imath} = \begin{bmatrix}1\\0\end{bmatrix}\) to \(\begin{bmatrix}0\\1\end{bmatrix}\) and \(\hat{\jmath} = \begin{bmatrix}0\\1\end{bmatrix}\) to \(\begin{bmatrix}-1\\0\end{bmatrix}\). Those two landing spots become the columns. Applying this matrix to any vector rotates it 90 degrees.

Python lesson-03-matrices.py 
# i_hat [1,0] lands at [0,1]
# j_hat [0,1] lands at [-1,0]
rotation_matrix = np.array([
    [0, -1],
    [1,  0]
])

test_vectors = [np.array([1, 0]), np.array([0, 1]), np.array([2, 3])]
for vec in test_vectors:
    result = (vec[0] * rotation_matrix[:, 0]) + (vec[1] * rotation_matrix[:, 1])
    print(f"Rotation: {vec} -> {result}")
Output
Rotation: [1 0] -> [0 1]
Rotation: [0 1] -> [-1  0]
Rotation: [2 3] -> [-3  2]

Part 3: Shear matrix

A shear keeps \(\hat{\imath}\) fixed at \(\begin{bmatrix}1\\0\end{bmatrix}\) but moves \(\hat{\jmath}\) to \(\begin{bmatrix}1\\1\end{bmatrix}\). The result: vertical lines tilt, horizontal lines stay flat. Space is slanted, not rotated.

Python lesson-03-matrices.py 
# i_hat stays fixed at [1,0]
# j_hat moves to [1,1]
shear_matrix = np.array([
    [1, 1],
    [0, 1]
])

for vec in test_vectors:
    result = np.matmul(shear_matrix, vec)
    print(f"Shear: {vec} -> {result}")
Output
Shear: [1 0] -> [1 0]
Shear: [0 1] -> [1 1]
Shear: [2 3] -> [5 3]

Part 4: Linearly dependent columns (space collapses to a line)

When column 2 is a multiple of column 1, any input vector gets squished onto that single line. All outputs are scalar multiples of each other. The transformation destroys one dimension of information.

Python lesson-03-matrices.py 
dependent_matrix = np.array([
    [1,  5],
    [3, 15]
])

# apply to several vectors and show they all land on the same line
for vec in test_vectors:
    result = np.matmul(dependent_matrix, vec)
    print(f"Squished: {vec} -> {result}")

# all results are scalar multiples of each other, all on the same line
Output
Squished: [1 0] -> [1 3]
Squished: [0 1] -> [ 5 15]
Squished: [2 3] -> [17 51]
AI Connection

How This Connects to AI

Why this matters for machine learning

Weight matrices in neural networks are linear transformations. Every layer in a neural network applies one of these: it takes an input vector and transforms it into a new vector in a different space. The columns of the weight matrix tell you where each basis direction lands.

When you write x @ W in PyTorch, that is exactly what we did with np.matmul(). x is your input vector or batch of vectors. W is the weight matrix, the transformation. The result is the transformed vector that flows into the next layer.

Stacking layers means applying multiple transformations in sequence. Each one rotates, scales, and stretches the data into a new space where patterns become easier to separate and classify. Deep networks are powerful because they can compose many simple linear transformations (plus nonlinearities) into very complex mappings.

Transformers are almost entirely matrix multiplications. The attention mechanism computes \(Q K^\top\), a matrix multiplication. The feedforward layers compute \(x W_1\) then \(x W_2\), two more. The embedding lookup is a matrix multiplication. Every single operation in a transformer reduces to this one operation applied in higher and higher dimensions.

The dependent-columns case shows a real failure mode. If a weight matrix has linearly dependent columns, it is rank-deficient: it collapses some dimensions and loses information. This is related to why regularization and careful initialization matter. You want your weight matrices to keep space full-rank so the network can represent rich transformations.
